## 核心需求

自动创建claude agent teams（）完成AI agent视频生成程序的设计、开发、测试和部署，并持续自动学习保持每周日23点更新一版

claude agent teams command：

```
$env:CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS = "1"
```

#### 功能一

1、自动从平台获取点赞数或浏览数或收藏数最高的视频下载本地作为原始视频；
2、自动对下载的原始视频进行二次创作生成新原创视频；
3、自动发布到其他平台

#### 功能二

1、自动解析分析用户的关键词（或文档或小说）生成创作视频剧本；
2、自动根据视频剧本生成创作分镜剧本；
3、自动根据分镜剧本生成首尾帧图片；
4、自动根据分镜剧本和首尾帧图片生成原创视频；
5、自动发布到其他平台

#### 功能补充

视频生成组件要包括2个选择，具体由用户手动录入；
1、主流AI模型平台和api，apikey用户来负责填写；
2、使用自建的n8n+comfyui+ollama，配置由用户填写；
3、所有的视频数据目录都要实时用oss命令同步到阿里云oss上，配置由用户配置；
4、所有的下载和上传操作都必须真人模拟操作，严格规避或绕开平台的风控规则

#### claude agent team计划说明

1、自动按照核心需求轨迹进化；
2、自动搜索github上最好的产品和kubegogo的仓库；
3、自动在kubegogo创建仓库并实时push项目 <https://github.com/kubegogo/genvideo.git> ；
4、使用minimax的模型，如果token提示消耗完了就等5小时恢复再继续
5、agent teams的除非非常必须的任务才等待否则尽可能并行进行

#### agentteams团队和技术选型

前端开发，技术选react；
后段开发，技术选golang，数据库选mysql8，缓存选redis7，自己本地部署；
测试者，技术选全套完整测试技术栈，问题直接提交kubegogo仓库

#### 环境说明

1、开发在本机环境进行
2、测试部署到docker中进行测试
3、mysql、redis在本机docker自建
4、程序模型调用优先使用国内
5、代码在 <https://github.com/kubegogo/genvideo.git>
6、程序所有相关信息放在docs目录
7、软件产品操作流程、程序核心功能架构和依赖、数据库字典都要生成写入docs目录
8、前端端口固定使用：3003、后端agent端口固定使用：3004 ，数据库、缓存或其他端口如有碰撞请绕开
